## 목업 실행 (현황판)

```bash
cd 개발/앱
npm install      # 최초 1회
npm run dev      # → http://localhost:5173/
```

> 무설정 실행을 위해 Tailwind는 Play CDN(`index.html`), 폰트는 인라인 `@import`(Pretendard·Nanum Myeongjo)로 로드합니다. 본개발에선 `tailwind.config` 빌드 + TS로 교체.
>
> **구현 범위 (전체 인터페이스 · 더미데이터)**
> - **관리자 권한**: 현황판 · 빈소·호실 상세 · 사이니지 · 예약·접수 · 편집·컨펌 · 콘텐츠 허브 · 외부 링크 · 정산(파트너사별/건별) · 환경설정(템플릿·AI프롬프트·BGM·알림·사용자권한·디바이스) · 시스템(잡 큐·업타임)
> - **파트너 권한**: 통합 대시보드 · 예약 접수 · 예약 목록 · 라이브 컨트롤 · 정산 조회 · 설정 (자사 테넌트 스코핑)
> - **유저 링크**: 모바일 7단계 위저드 (동의→업로드→전환→음악→편지→미리보기→완료)
> - **영상 편집기**: 좌 블록리스트(순서 고정) / 중앙 미리보기(서버 렌더 재생)+트림+타임라인 / 우 블록타입별 편집 패널 — 상단 마스터 네비의 "영상 편집기" 또는 각 화면의 "컨펌/편집기 열기"로 진입
>
> 디자인은 `디자인_가이드.md`(아이보리/네이비/골드·명조·빨강 배제·lucide), IA는 `메뉴구조_IA.md`를 따랐고, 시뮬레이터(`참고자료/`)는 기능·플로우 디테일만 참고했습니다. `현황판_목업.jsx`는 최초 단일파일 목업 원본으로 보존(앱은 이를 컴포넌트로 분리·확장).

## IDE/Claude Code에서 이어가는 법

1. **먼저 읽기**: `개발/개발착수전_정리.md`(결정) → `개발/기술명세서_v3.md`(스펙) → `개발/디자인_가이드.md`(톤) 순. (배경 맥락은 `참고자료/프로젝트_맥락_요약.md`)
2. **프론트 시작**: 현재 `개발/앱`은 무설정 실행용. 본개발은 Vite + React(**TS**) + Tailwind(빌드)로 올리고 `현황판_목업.jsx`를 컴포넌트로 분리 이식 (디자인 가이드의 "프론트 이관" 절 참고).
3. **로고**: 가능하면 진짜 알파 채널 살아있는 원본 PNG/SVG를 받아 `src/assets`에 넣고 import (지금 b64는 임시).
4. **폰트**: `pretendard` npm 패키지 + Nanum Myeongjo(고인 성함용) 설치.
5. **다음 화면**: 영상 편집기 + 컨펌 화면 (좌 블록리스트 / 중앙 미리보기=서버 렌더 영상 재생 + 트림 / 우 블록별 편집패널).
6. **백엔드**: 명세서·정리.md의 스택대로. 목업의 더미 데이터(ROOMS)를 API 연결로 교체.

> 주의: 이 목업은 "화면 구조·톤 확정"이 목적이라 단일 파일·CDN 폰트·더미 데이터입니다. 본개발에선 구조·디자인은 그대로 살리고 데이터 연결·컴포넌트 분리만 추가하면 됩니다(화면 다시 안 만듦).
