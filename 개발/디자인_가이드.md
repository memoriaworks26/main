# 디자인 가이드 — Memoriaworks 관리 시스템

목업(`현황판_목업.jsx`)에 적용된 디자인 규칙과, 이를 본개발 프론트로 옮기는 방법.

## 톤 방향

- **장례 도메인 운영 솔루션**. 신식 트렌디 SaaS가 아니라, 기존 장례식장 운영자(funein·기품 사용층)에게 **익숙하고 안정적인 업무 도구** 톤.
- 레퍼런스: **funein** 구조(좌측 다크 사이드바 + 상단 탭 + 카드 그리드, 고정폭/비반응형) 차용.
- "AI가 찍어낸 제네릭"(보라 그라데이션·Inter·과한 라운드·장식 애니메이션) 회피.
- 세련됨 = 절제·정밀(Linear/Stripe/Tufte data-ink). 색은 의미(상태)에만, 여백·헤어라인·서체 위계로 품격.
- **시그니처**: 고인 성함을 명조(세리프)로 → 로고 세리프와 호응, 위패·지방의 전통 결.

## 서체

- **UI 전체**: Pretendard (`pretendard` npm 또는 CDN). 폴백: system 고딕.
- **고인 성함(시그니처)**: Nanum Myeongjo (명조). 폴백: Apple Myungjo / Batang / serif.

## 규칙

- 모서리 radius 3~4px (과한 라운드 금지), 그림자 대신 **헤어라인 경계**.
- 상태 태그는 각진(둥근 알약 X) + 색점.
- 스피너 대신 **얇은 진행 바**(제작중). 모션은 상태 표시에만, `prefers-reduced-motion` 존중.
- 상단 **조용한 요약 스트립**(진행/컨펌대기/제작중/발행완료) — 큰 KPI 박스 금지(헤어라인 인라인 숫자).
- **고정폭, 비반응형**(PC 운영자 화면). 편집기도 PC 전용.

## 목업 현재 구성

- **마스터 네비**(최상단): 관리자 권한 / 파트너 권한 / 유저 링크 — 탭 클릭 전환. (추후 권한 분리)
- **관리자 권한**: 사이드바(로고+빈소 목록+메뉴) + 현황판(요약 스트립 + 빈소 카드 그리드). → 완성
- **파트너 권한 / 유저 링크**: 자리표시(구성 예정 안내).
- ⚠️ **사이드바 메뉴 IA는 아직 funein 가안**(콘텐츠허브·정산·환경설정 등). 우리 설계 IA로 교체 필요:
  현황판 / 제작·편집 / 컨펌 대기 / 발행·링크 / 콘텐츠 허브 / 사이니지 / 정산 / 파트너사·권한 / 설정.
  (funein류 운영기능 — 위생처리관리대장·지방·축문·화장예약 등 — 은 이번 범위 제외)

## 본개발 프론트로 이관하는 법

목업은 화면·톤 확정용 단일 파일이다. 본개발에선 **구조·디자인은 그대로 살리고** 아래만 추가:

1. **프로젝트**: Vite + React(TS) + Tailwind.
2. **디자인 토큰**: 위 색·서체를 `tailwind.config` (theme.extend.colors) + CSS 변수로 정리.
3. **컴포넌트 분리**: `MasterNav`, `AdminView`, `RoomCard`, `Slate`, `Tag`, `NavItem`, `Summary`, `Placeholder` 를 각 파일로.
4. **로고**: 실제 파일(투명 PNG/SVG)을 `src/assets`에 두고 import. (지금 b64는 임시)
5. **폰트**: `pretendard` 패키지 import, Nanum Myeongjo 웹폰트.
6. **데이터**: 더미 `ROOMS` → 백엔드 API 연결. 컴포넌트는 그대로, props 데이터만 실데이터로.
7. **라우팅**: 마스터 네비 탭 → 실제 권한별 라우트/가드로. (추후 권한 분리)
8. **다음 화면**: 영상 편집기 + 컨펌 (좌 블록리스트(순서고정) / 중앙 미리보기=서버 렌더 영상 재생 + 트림 슬라이더 / 우 블록타입별 편집패널).
