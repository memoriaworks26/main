# 사이니지 라즈베리파이 — 쉬운 설명서

> 호실 TV마다 붙는 작은 컴퓨터(라즈베리파이) 이야기. 비기술자도 읽을 수 있게 쉬운 말로 정리.
> 작성: 2026-06-25 · 아직 **개발 시작 전**(서버·콘솔 쪽은 거의 준비됨, 파이 쪽은 0)

---

## 한 줄 요약

호실 TV마다 뒤에 **작은 컴퓨터(라즈베리파이) 한 대**를 붙인다.
이 컴퓨터의 일은 딱 하나 — **"이 방에서 지금 틀어야 할 영상"을 알아서 받아다 트는 것.**
직원은 손 안 댄다. 말하자면 **각 방 전담 영상 직원**(똑똑한 DVD 플레이어).

---

## 평소엔 이렇게 돌아간다

```
파이가 서버한테 3초마다 물어봄
   "나 1호실인데, 지금 뭐 틀어?"
        ↓
서버: "콩이 추모영상 틀어" + 영상 위치(주소) 알려줌
        ↓
파이가 그 영상을 받아서 → TV에 재생
```

핵심: 영상을 **파이가 들고 있는 게 아니라**, 서버가 매번 "이거 틀어" 하고 알려준다.
그래서 영상이 바뀌어도(식마다 다른 펫) 파이는 손댈 게 없다.

---

## 필요한 기능 (v1)

### ① 영상 1건 받고 → 틀고 → 끝나면 지우기
- **"미리 받아두고, 끝나면 지운다"** 가 핵심.
- 다음 식 영상은 **현재 식 재생 중에 미리** 받아둔다. (식과 식 사이에 화면이 비지 않게)
- 끝난 영상은 바로 삭제 — 용량 절약 + 개인정보(고인·가족 영상) 보호.
- 들고 있는 건 보통 *대기화면 + 현재 영상 + 다음 영상* 정도. 누적 안 함.

### ② USB 꽂으면 자동 재생 (비상수단)
- 인터넷이 죽어도 직원이 **USB 하나 꽂으면 바로 그 영상**이 나온다.
- 안전장치 2가지:
  - **영상 파일만** 튼다. (USB 속 프로그램·스크립트는 절대 자동 실행 안 함)
  - **USB를 빼면** 자동으로 원래(서버 명령) 모드로 복귀.
- USB가 꽂혀 있는 동안은 서버 명령보다 USB가 우선.

### ③ 문제 생기면 스스로 복구
- 리부팅은 **최후의 수단**(껐다 켜면 1분 검은화면이라 유가족 앞에선 별로).
- 순서: **영상만 슬쩍 다시 켜기 → 그래도 안 되면 → 리부팅.**
- 그리고 **무슨 일이 있어도 화면엔 영상 아니면 대기화면만.** (아래 ★ 참고)

### ④ 첫 부팅 때 자동 설치
- 메모리(SD카드)에 **"이건 1호실" 정보를 미리 구워서** 출고.
- 직원은 **전원만 꽂으면 끝.** 파이가 첫 부팅에 알아서 "나 1호실" 등록하고 평소 모드로 들어감.
- (대안) 모니터에 페어링 코드/QR 띄우고 관리자 콘솔에서 호실 연결.

### ★ 절대 검은화면 금지 (가장 중요)
- 부팅 화면 → 대기화면 → 영상. 그 사이 **검은화면·에러·컴퓨터 메뉴·마우스 커서 절대 노출 금지.**
- 뭐가 잘못돼도 화면엔 **항상 영상 아니면 대기화면.**

### 그 외 꼭 필요한 것
- **"나 살아있어요" 신호(하트비트)** — 파이가 서버에 주기적으로 신호를 보내야, 관리자 화면에서 *1호실 온라인 / 3호실 꺼짐* 이 진짜로 보인다.
- **시계 자동 맞추기** — 파이는 전원 끄면 시계가 틀어진다. 시간이 틀리면 "오늘 예약"을 잘못 골라 엉뚱한 영상을 틀 수 있어, 부팅 때 인터넷으로 시간을 맞춘다.
- **원격으로 고치기** — 나중에 프로그램 고칠 일이 생겨도 매번 현장에 안 가도 되게, 원격 업데이트를 처음부터 넣어둔다.

| 항목 | 결정 | 이유 |
|---|---|---|
| 영상 재생 방식 | **네이티브 mpv** | 끊김 없는 무한반복·안정성. 유가족 앞 추모영상이 제일 단단해야 함 |
| 하드웨어 | **라즈베리파이 4 (4GB) + USB SSD** | 1080p 안정 재생 + 정전 시 메모리 손상 회피 |

| 영상 주소 | **서버가 매번 새로 발급** | 식마다 영상이 다르고, 주소는 만료됨. 파이는 영상번호로만 캐시 |

## 조직 구조 — 디바이스는 4단계 소속

```
사업부(biz_units) ← 파트너사(partners) ← 호실(rooms) ← 디바이스(signage_devices)
```

- 디바이스는 `partner_id` + `room_id`를 가지고, **사업부는 파트너를 타고 자동으로 도출**(별도 컬럼 없음).
- DB 접근 격리(RLS)는 이미 이 3단계대로 적용됨 — 직원은 자기 사업부 디바이스만 본다.
- 그래서 **디바이스 등록은 3단계로 고른다**: 사업부 → 파트너사 → 호실.
- 관리자 콘솔 사이니지 화면은 지금 *파트너사 1단계*로만 거를 수 있어, *사업부→파트너사→호실* 캐스케이드 필터를 더해야 한다.

## 영상이 정해지는 길 (서버 쪽 — 이미 준비됨)

```
이 파이(1호실) → 그 호실의 오늘 예약 → 그 예약의 발행된 추모영상 → 주소 발급
signage_devices → rooms ← reservations ← videos(final_path)
```

서버는 이 연결을 따라가 **"지금 이 방 영상"의 새 주소**를 파이에 내려준다.
이 길은 DB에 이미 다 깔려 있고, 손볼 곳은 2군데:
1. 디바이스↔호실 연결을 **호실 이름(텍스트) → 호실 ID(고정값)** 로 정리.
2. "현재 영상" 고르는 규칙 확정(*오늘 예약 + 발행됨 + 안 만료*).

## 라이브 컨트롤 — 파이가 따라야 할 것

관리자/파트너의 [라이브 컨트롤](../앱/src/partner/live.jsx) 화면이 누르는 걸 파이가 그대로 따라야 한다.

- **상태(폴링으로 해결됨)** — 모드(광고/대기/알림/제작영상), 음량, 음소거, 소스 선택, 알림문구. DB에 저장돼 있어 파이가 3초마다 읽어 적용.
- **명령(빈 곳 — 채워야 함)** — "플레이어 재시작 / 강제 새로고침 / 강제 재다운로드"는 *일회성 명령*이라 상태 폴링만으론 안 됨. → `signage_devices.pending_cmd` 같은 **명령 칸**을 두고, 파이가 읽어 실행한 뒤 비운다. (지금은 toast 목업)
- **재생/정지** — 현재 화면 메모리에만 있고 DB 미반영 → DB에 저장해야 파이가 안다.
- **모드별 렌더** — 제작영상뿐 아니라 광고/대기/알림 소스 영상·이미지 + 알림 문구(서버가 예약값 치환한 최종 텍스트)까지 파이가 받아 그린다. (mpv라 자막 오버레이는 파이가 직접)

## 작업계획 (4단계)

> 의존성 순서: 서버 먼저(파이가 물어볼 상대가 있어야 함) → 파이 최소재생 → 무인 안정성 → 배포

### 1단계 — 서버 토대 (파이 없이도 테스트 가능)
1. **마이그레이션 0033** — 디바이스 토큰 컬럼 + 디바이스↔호실 ID 정리(텍스트→고정ID) + 명령 칸 `pending_cmd`
2. **`device-sync`** — 토큰 확인 → 호실→오늘예약→발행영상 → 새 영상주소 + 모드별 소스 + 알림문구 내려줌 → last_comm 기록 → 대기 명령 전달
3. **`device-enroll`** — 등록코드 → 디바이스 토큰 발급
- ✅ 끝나면: 파이 없이 curl로 "1호실 영상 주소가 제대로 나오는지" 확인

### 2단계 — 파이 최소 재생 ("검은화면 금지" 첫 데모)
4. 부팅 → 전체화면 + 커서·부팅로그 숨김 + 대기화면 상시
5. 3초 폴링 루프(device-sync)
6. 영상 받아 mpv 무한반복, 영상번호 바뀌면 교체
- ✅ 끝나면: 진짜 호실 영상이 TV에 뜸 (토큰 하드코딩)

### 3단계 — 무인 안정성 (장례식장 필수)
7. 미리받기 + 끝난 영상 삭제(현재+다음 롤링)
8. 자동복구(플레이어 재시작 → 안 되면 리부팅)
9. 검은화면 금지 보장(어떤 에러든 대기화면)
10. 하트비트(last_comm) → 콘솔 온/오프라인 실값
11. NTP 시간동기
12. 모드별 렌더(광고/대기/알림 + 알림 오버레이) + 명령 실행(재시작/새로고침)
- ✅ 끝나면: 전원만 꽂아두면 무인으로 돌고, 관리자가 멀리서 상태·제어

### 4단계 — 배포·운영 마감
13. USB 비상재생(영상만·noexec, 빼면 복귀)
14. 첫 부팅 자동등록(SD에 호실정보 굽기 → device-enroll)
15. 원격 업데이트
16. 콘솔 마감 — 사이니지 콘솔에 *사업부→파트너사→호실* 캐스케이드 필터 + 라이브 컨트롤 toast 목업을 실제 명령으로 연결 + 재생/정지 DB 반영
- ✅ 끝나면: SD 구워 현장에 꽂으면 끝. 실서비스 가능

**시작점:** 1단계(서버). 파이 하드웨어 없이 노트북에서 만들고 검증 → 파이 도착하면 2단계부터.

## 추가 점검 (v1 필수 ★)

위 4단계와 별개로, 빠지면 현장에서 터지는 것들. 해당 단계에 끼워 넣는다.

### 보안·개인정보 (추모영상 = 고인+가족 PII)
- [ ] **캐시·토큰 암호화** — 영상 캐시·디바이스 토큰을 SD/SSD에 평문 저장 금지(분실·도난 시 유출). 캐시 파티션 암호화 + 토큰 권한 잠금
- [ ] **토큰 폐기(revoke)** — 디바이스 분실/교체 시 토큰 무효화 → device-sync 거부 (지금은 발급만 있음)
- [ ] 파이 로그·에러에 보호자명·영상경로 등 PII 안 남기기

### 화면·하드웨어
- [ ] **화면 방향·화면비** — 모니터가 세로(포트레이트)일 수 있음. 가로/세로 + 레터박스/크롭 설정
- [ ] **한글 폰트** — mpv 자막 오버레이용 한글 폰트 설치(안 하면 알림문구 □□□)
- [ ] HDMI-CEC / TV 전원 연동 / EDID 해상도 자동인식 (TV 껐다 켜면 신호 자동복구)
- [ ] 24시간 가동 방열 케이스 + 정품급 전원 어댑터(전원 부족 = 불안정 1순위)
- [ ] 정전 복구 시 자동 부팅→재생(부트 설정)

### 운영·장애대응
- [ ] **오프라인 자동 알림** — 일정시간 죽으면 이메일/카톡 알림(기존 send-email 함수 활용). 콘솔 표시만으론 아무도 안 봄
- [ ] 원격 접속·중앙 로그(현장 안 가고 진단) — 원격 *업데이트*와 별개
- [ ] 예비장비 핫스왑(스페어 SD 꽂으면 그 호실로 복구) + 설치 가이드 1장 + 호실 라벨
- [ ] device-sync 가볍게(파이 수십~수백 대 × 3초 폴링 부하 대비, 필요시 간격 조정)

### 재생 엣지케이스 (장례 특수상황)
- [ ] **하루 2식 / 식 사이** — slot 기준 전환, 식 사이 청소시간엔 대기화면
- [ ] **영상 아직 안 나왔는데 식 시작**(렌더 중) → 대기화면 폴백(검은화면 X)
- [ ] 식 도중 영상 재발행(수정본) → video_id 바뀔 때 교체 타이밍 규칙(즉시 vs 한 바퀴 후)
- [ ] 예약 변경·취소 → 다음 폴에 자동 반영되는지 확인

### 콘솔·비용
- [x] **디바이스 등록 UI** — 콘솔에서 *사업부→파트너사→호실* 고르고 등록코드 발급 (관리자 사이니지 화면, 2026-06-25 완료)
- [ ] 다운로드 비용(egress) — 캐시로 재다운 방지 + 가능하면 egress 0인 R2(`storage_provider` 이미 있음)
