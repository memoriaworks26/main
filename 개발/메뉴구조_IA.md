## 0. 설계 원칙

1. **권한이 1차 축, 작업 흐름이 2차 축.** 상단 마스터 네비로 권한 트랙(관리자/파트너/유저)을 전환하고, 각 트랙 안에서 좌측 사이드바가 작업 흐름순으로 묶인다.
2. **제작 파이프라인을 그대로 메뉴화.** `폼 접수 → 자동 렌더 → 편집·컨펌 → 발행 → 사이니지 표출`이 메뉴 동선과 1:1 대응.
3. **테넌트 스코핑.** 파트너(장례식장)는 자사 빈소·예약·영상만. 관리자(사업부)는 전 파트너사 + 시스템 설정.
4. **운영/시스템 메뉴 분리.** 잡 큐·에러 모니터(BullBoard·Sentry)는 일상 운영과 섞지 않고 총관리자 전용 그룹으로.
5. **색은 상태에만, 장식 배제** (목업 디자인 가이드 유지). 메뉴는 아이콘+라벨 헤어라인 톤.

## 1. 마스터 네비 (상단 · 권한 전환) — 기존 유지

| 트랙 | 대상 | 범위 | 디자인 트랙 |
|------|------|------|-------------|
| **관리자 권한** | 사업부 / 총관리자 | 전 파트너사·전 호실 + 시스템 설정 | 운영툴(조종석) |
| **파트너 권한** | 장례식장 직원 | 자사 빈소·예약·영상만(멀티테넌트 스코핑) | 운영툴(관리자 축소판) |
| **유저 링크** | 유족·조문객 | 발행 영상 1건(모바일 공개) | 위로 중심·별도 톤 |

> 권한 계층(명세서 §8): 사업부(총관리자) → 파트너사(자사 범위 제한). 파트너 화면 = 관리자에서 기능을 떼어 구성.

## 2. 관리자 사이드바 (funein 가안 교체안) ★

```
관리자 권한
│
├─ 운영
│   ├─ 현황판                  전 호실 상태 그리드(제작중/컨펌대기/발행완료/예비/안내)
│   ├─ 빈소·호실 ▸             장소별 진행 영상 + 사이니지 상태
│   │     1호실 (1층) · 2호실 (1층) · 3호실 (2층) · 4호실 (2층)
│   │     예비 · 종합안내 로비 · 종합안내 현관
│   └─ 사이니지                디바이스(라즈베리파이) 매핑·재생·온라인 상태   ⚠️E
│
├─ 추모영상 제작
│   ├─ 예약·접수               영상 목록(상태별 필터) · 폼 접수            ⚠️D
│   ├─ 편집·컨펌               블록 편집기 + 미리보기(서버렌더 재생) + 컨펌  ★다음 우선순위
│   ├─ 콘텐츠 허브             클립·사진 선업로드 자산 풀
│   └─ 외부 링크               발행 HLS 링크·토큰·만료 관리
│
├─ 정산
│   └─ 정산 내역               단가 스냅샷·거래명세서(발행 시 동결)·메일 발송
│
├─ 환경설정 (관리자 전용)
│   ├─ 템플릿                  파트너사별 블록 순서·구성(고정 규칙)
│   ├─ AI 프롬프트             리스트 선택형 프롬프트 관리(타이틀·AI영상)
│   ├─ BGM 라이브러리          곡 등록(관리자는 곡 교체만 가능)
│   ├─ 알림                    발신번호·알림톡 템플릿(임계경로 ⚠️)
│   ├─ 사용자·권한             파트너사(테넌트)·계정·역할
│   └─ 디바이스 등록           라즈베리파이 ID·호실 매핑(udev/systemd)
│
└─ 시스템 (총관리자 전용)
    ├─ 잡 대시보드             렌더 큐 3종 상태·실패 잡 재처리(BullBoard)
    └─ 에러·업타임             Sentry·외부 의존성(OpenAI·Kling·솔라피) 모니터
```

**상단바(헤더) 공통**
- 검색 / 전역 알림(Bell): 컨펌 요청·렌더 실패·큐 적체 등 액션 알림.

## 4. 유저 링크(모바일) 화면 구조

단일 화면 트랙(사이드바 없음):
- 발행 추모영상 재생(HLS) — 차분·정중 톤(아이보리/웜그레이)
- 토큰 링크(퇴실 시 즉시 무효화)
- 위로 중심으로 운영툴과 톤 분리(별도 디자인 트랙)

> **⚠️D 유저 주체 미확정**이 이 트랙의 범위를 좌우: 유족 직접 입력이면 **폼·업로드 화면**이 유저 트랙에 추가되고, 직원 대행이면 재생 전용으로 남는다. → 메모리아웍스 확인 후 확정.

## 5. 미확정 항목이 IA에 주는 영향

| 항목 | IA 영향 | 분기 |
|------|---------|------|
| **⚠️D 유저 주체** | `예약·접수` 폼의 공개 위치 | 유족 직접 → 유저 트랙에 폼 추가 / 직원 대행 → 관리자·파트너 내부 폼만 |
| **⚠️E 사이니지 범위** | `사이니지` 메뉴 깊이 | 영상 루프만 → 디바이스 모니터 1뎁스 / 정보안내 화면도 → 안내 콘텐츠 편집(funein 협의·추가 견적) |

두 항목 확정 전까지 해당 메뉴는 **자리표시(구성 예정)** 로 두고 골격만 노출.

## 6. 목업 반영 메모 (현황판_목업.jsx 교체 지점)

- **교체 대상**: `현황판_목업.jsx`의 사이드바 `<nav>` 영역 — 현재 `현황판 / NAV_ROOMS / 콘텐츠 허브 / 정산 / 환경설정`(L216~223).
- **교체 방식**: 위 §2 트리대로 그룹 헤더(운영 / 추모영상 제작 / 정산 / 환경설정 / 시스템) + `NavItem` 재구성. `NAV_ROOMS`(L166)는 `빈소·호실` 하위로 그대로 유지.
- **신규 메뉴 6종**(사이니지 / 예약·접수 / 편집·컨펌 / 외부 링크 / 시스템 그룹 2종)은 우선 `Placeholder`(구성 예정)로 스텁, `편집·컨펌`을 다음 실제 화면으로 제작.
- 디자인 토큰·톤은 목업 그대로 유지(아이보리+네이비/앰버, 명조 시그니처, 상태색).
