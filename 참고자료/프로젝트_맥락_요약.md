# 프로젝트 맥락 요약 — 대화에서 확정된 결정

`01_기획/메모리아웍스_개발착수전_정리.md`가 마스터 문서. 이 파일은 그 위에 대화 전체 맥락을 요약한 보조 문서.

## 개요
- 고객사 메모리아웍스 / 공급자 노드앤드(박용진). 파트너사별 템플릿으로 추모영상 자동 제작→컨펌→발행, 사이니지(라즈베리파이) 표출.
- 견적 4,750만원(VAT별도). 일정 6/28 프로토타입 / 7/12 완료(압축 일정 — 안정성 검증 시간이 최대 리스크).
- 핵심 원칙: ① 안정성 1순위(비용 차순위, 서버 월 100만까지 가능) ② 관리형 인프라 ③ 디바이스 네트워크 독립.

## 기술 스택 / 인프라 (확정)
- 프론트: Vite + React(TS) + Tailwind, 정적 SPA. 편집기 PC 전용·비반응형.
- API: Node.js + NestJS. 워커: BullMQ + FFmpeg 자체구축(외부 렌더API 미사용), 분리 서비스·레플리카 2.
- **컴퓨트 = Render** (Railway에서 전환: 최근 6개월 주요장애 5건·8시간 전면장애로 안정성 감점. 개발 전이라 전환비용 0). 비상구 = 동급 PaaS 또는 AWS 서울(2차).
- DB·인증 = Supabase 서울 (Auth→NestJS RBAC·테넌트스코핑→RLS 2차). 큐 = Upstash(Redis, 컴퓨트 밖). 스토리지 = Cloudflare R2(egress 0, 서명URL). CDN = Cloudflare.
- 문자 = 솔라피(알림톡+SMS폴백). 이메일 = Resend. 운영 = BullBoard·Sentry·업타임.
- 디바이스 = 라즈베리파이4 + Pi OS Lite, mpv 루프, 네트워크 독립.
- **데이터·큐 전부 컴퓨트 밖** → Render가 죽어도 데이터 손실 0, 발행영상은 R2+CDN·사이니지 로컬재생으로 장애 중에도 재생 지속.

## 영상 제작/편집 모델
- 템플릿(블록 순서 고정) → 유저폼 입력 시 EDL 인스턴스 생성 → 미리보기·최종렌더가 동일 EDL 단일소스 참조.
- 블록: 타이틀(GPT 이미지→영정틀 오버레이→영상화) / 클립(콘텐츠허브) / 슬라이드(FFmpeg 합성) / AI영상(Kling) / 편지(FFmpeg+텍스트).
- 편집기 = 블록 기반 경량(트림 포함), 풀 NLE 아님. **미리보기 = 서버 렌더 영상 재생**(오버레이 이미 렌더에 포함 → Canvas 불필요). 소스변경 시 해당 블록만 재렌더. 컨펌 시 섹션별 렌더 → 컨펌 후 통합 렌더.
- 프록시: 업로드 시 480p 자동생성 + 정규화(HDR→SDR·VFR→CFR). 편집기는 프록시 재생, 최종만 원본.
- 어드민은 블록 내부 소스만 수정(교체·재생성·텍스트·트림), 순서 변경 안 함.

## 영상·출력 사양
- 출력: HLS(고객링크) + USB용 MP4. 화면비 **16:9 고정**(설정값화로 추후 9:16 확장 가능). 불일치 소스는 잘림없이 전체표시+블러배경.
- 오디오: 전체 1트랙 BGM + 페이드, 편지=배경음만, 유저원본 Mute. 관리자는 BGM 곡 교체만.
- 컨펌: 전건 수동. 정산: 단가 스냅샷 고정 + 발행 명세서 동결. 길이·개수 하드캡 없음 + 안정성 가드.
- **AI: GPT(타이틀)·Kling(AI영상) 고정** — 어댑터로 교체 가능, 런타임 선택 UI 없음. 프롬프트 = 리스트 선택형.
- 산출물 정리: 최종본·원본만 보관, 중간 산출물 7~14일 후 R2 자동삭제.

## 안정성 / 디바이스
- 대용량 동시 렌더(1~2GB×10): 큐 동시성 상한 + 워커 디스크 사전점검·즉시정리 + 워커 수평/수직 확장. **분업: 자원크기=서버단(Render 설정) / 안전로직(동시성·디스크가드·타임아웃)=개발단(코드, 환경변수)**. 정확한 수치는 프로토타입 실측.
- 라즈베리파이 24h 무중단: **소프트웨어 무중단=개발 범위**(읽기전용FS·워치독·예약재부팅·끊김없는 전환·장애시 직전영상 유지). **하드웨어 무중단=메모리아웍스 영역**(냉각·예비기기·선플래시SD·원격모니터링). 계약에 "하드웨어 고장은 개발사 책임 외" 명시.

## 비용 / 계약
- 운영비(AI·문자·인프라) 운영사 부담, 개발대금 별개. "무관리" 문구 명시 안 함.
- funein류 운영기능(지방·축문·문서출력·화장예약·위생대장 등) 이번 범위 제외, 추가 시 별도견적.
- 펫 장례 시스템 재사용으로 영상·AI·렌더 공수 절감 여지(IP 보호 계약조항 필요).
- 계약 핵심: 범위명시+제외, AI 비보장 면책, 외부의존성·플랫폼장애 면책(무중단 SLA 보장 안 함 명시), 운영사 귀책지연 면책, 검수기준, IP 재사용모듈 보호, 개인정보 수탁지위, 책임한도.
